Сценарии:
1. 100 виртуальных машин на базе Linux и Windows, общие задачи, нет особых требований. Преимущественно Windows based-инфраструктура, требуется реализация программных балансировщиков нагрузки, репликации данных и автоматизированного механизма создания резервных копий.
Решение:
vCenter + ESXi + Veem Bcakup
Выбор решения основываеттся на практике, использовали различныке решения, данный вариант самый предпочтительный. Имеется встроенный механизм балансировки DRS, интеграция с AD, так же встроенный мехпнизм  репликации и бэкапов. Имеется API для работы с Veem Bcakup, эта связка налету бэкапит и восстанваливает полностью  ВМ, но и  данные с ВМ.

2. Наиболее производительное бесплатное open source-решение для 20-30 серверов.
Решение:
Proxmox VE. Бесплатен, прост в настройке и управлении. Идеален для небольшого количества ВМ. Так же  имеется возможность создания кластера HA и системы СРК на базе  одного вендора.

3. Бесплатное, совместимое и производительное решение для Windows-инфраструктуры

Решение:
Microsoft Hyper-V Server . Бесплатен полная совместимость с Windows Инфраструктурой, один Вендор.
