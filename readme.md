Задачи:

1. Высоконагруженная база данных MySql, критичная к отказу.

Тип платформы: Предпочтительно использовать физические серверы, объединённые в HA-кластер (СУБД требовательны к производительности дисковой системы, при использовании физических серверов идёт прямое обращение к дискам). Также можно использовать паравиртуализацию, но на мощном железе корпоративного класса, применяя сопутствующие настройки и драйверы.

2. Различные web-приложения.

Тип платформы: Виртуализация уровня ОС (Веб-приложения в настоящее время используют микросервисный подход, а не монолит. Микросервисы позволяют решить вопрос с зависимостями). Различные веб-приложения разворачивать как отдельный сервис в контейнере. Можно использовать и паравиртуализацию, если ограничены в серверах, развернув отдельную ВМ, на которой запущены контейнеры с веб-приложениями.

3. Windows-системы для использования бухгалтерским отделом.

Тип платформы: Паравиртуализация.

4. Системы, выполняющие высокопроизводительные расчёты на GPU.

Тип платформы: Физические серверы, простота настройки и максимальная производительность. Пробовали различные кейсы на практике — самый подходящий вариант.
