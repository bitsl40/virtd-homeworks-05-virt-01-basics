Задачи:
1. Высоконагруженная база данных MySql, критичная к отказу.
  Тип платформы: Предпачтительно использовать физические сервера объедененные в HA кластер(СУБД требовательны к производительности дисковоцй системы,   при использваонии   фифзических серверов идет прямое обращение к дискам), так же можно использовать паравиртуализацию но на мощном железе корпоративного класса, применяя сопутсвующие настройки и драйвера. 
2. Различные web-приложения.
   Тип платформы: Виртуализация уровня ОС (Веб приложения в настоящее время используют микросерисный подход , а не монолит. Микросервисы позволяют решить вопрос с зависимостями). Различные Web приложения разворачивать, как отдельный сервис в контейнере. Можно  использовать и паравиртуализацию, если  ограничены в серверах, развернув отдельныую ВМ на которой запущены контенеры с Web приложениями.
3. Windows-системы для использования бухгалтерским отделом.
   Тип платформы: Паравиртуализация
4. Системы, выполняющие высокопроизводительные расчёты на GPU.
   Тип платформы: Физические сервера, простота настройки и максимальная производительность. Пробовали различные кейсы на практике, самый подходящий вариант.
